Pourquoi Task API

🚀

Démarrage instantané

Un seul docker compose up et l'API tourne sur le port 8000 avec migrations automatiques.

📦

CRUD complet

Créez, lisez, mettez à jour et supprimez des tâches avec une pagination intégrée (20 items/page).

🔐

Auth JWT

Inscription, login et refresh token. Chaque utilisateur ne voit que ses propres tâches.

Référence

Endpoints disponibles

POST /api/auth/register/ Créer un compte
POST /api/auth/token/ Obtenir access + refresh
POST /api/auth/token/refresh/ Rafraîchir le token
GET /api/health/ Vérifier que l'API répond (public)
GET /api/tasks/ Lister ses tâches (auth requise)
POST /api/tasks/ Créer une tâche (auth requise)
GET /api/tasks/{id}/ Détail d'une tâche
PATCH /api/tasks/{id}/ Mise à jour partielle
DELETE /api/tasks/{id}/ Supprimer une tâche

Quick start

Lancer en 30 secondes

terminal 
git clone https://github.com/clems023/task-api.git
cd task-api
cp .env.example .env
docker compose up --build
curl — authentification + tâche 
curl -X POST http://localhost:8000/api/auth/token/ \
  -H "Content-Type: application/json" \
  -d '{"username":"alice","password":"securepass123"}'

curl -X POST http://localhost:8000/api/tasks/ \
  -H "Authorization: Bearer <access>" \
  -H "Content-Type: application/json" \
  -d '{"title": "Ma première tâche"}'
